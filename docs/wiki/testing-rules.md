# Testing Rules

## 기본 원칙

- 엔진, 스토어, 페이지는 각각의 성격에 맞게 테스트한다.
- 동작 규칙이 바뀌면 UI만 고치지 말고 엔진 또는 스토어 테스트도 같이 본다.
- 위키 문서가 테스트보다 우선하지는 않는다. 테스트와 실제 동작이 최종 사실이다.

## 테스트 위치

- 엔진: 각 feature 폴더의 `*.test.ts`
- 컴포넌트/페이지: 각 feature 폴더의 `*.test.tsx`
- 라이브러리: `src/lib/*.test.ts`

## 권장 검증 순서

1. 관련 feature 문서 읽기
2. 기존 테스트 확인
3. 구현 변경
4. 영향받는 테스트 수정 또는 추가
5. `npm test` 또는 최소 관련 테스트 실행

## 변경 유형별 테스트 기준

### 저장 구조 변경

- parse/load/save 경로 테스트
- 마이그레이션 테스트
- import/export 테스트

### 세션 흐름 변경

- 엔진 또는 스토어 테스트 우선
- 페이지는 핵심 인터랙션만 보강

### 목록/검색/필터 변경

- 렌더링 결과
- 숨김 동작
- 즐겨찾기와 검색 결합

### 성능성 UI 조정

- 재마운트 회피 여부
- `useDeferredValue`, `startTransition` 같은 흐름이 유지되는지 확인

## 수동 확인이 필요한 영역

- QR 카메라 스캔
- 실제 브라우저의 IndexedDB 동작
- 디렉터리 접근 기반 에디터 저장
- CSS 체감 성능

## 실행 명령

```bash
npm test
```
