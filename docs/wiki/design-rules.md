# Design Rules

## 최상위 원칙

- 미니멀리즘 우선
- 아이콘 중심 UI 우선
- 텍스트 설명 최소화
- 사용자는 설명 없이도 기능 의미를 추론할 수 있다고 가정

## 메인 앱 규칙

- 홈은 큰 액션 카드와 짧은 캡션만 쓴다.
- 설명 문구가 길어지면 먼저 아이콘, 배치, 상태 노출로 해결한다.
- 세션 이어하기, 결과 다시 보기 같은 중요한 상태는 배너로 즉시 노출한다.

## 목록과 편집기 규칙

- 선택 상태와 주요 조작은 표나 카드 표면 안에서 바로 드러나야 한다.
- 별도 우측 상세 패널 의존을 피한다.
- 카드 숨김은 실제 텍스트 감춤이 아니라 명확한 마스킹 표현을 유지한다.

## 에디터 규칙

- 모바일을 고려하지 않는다.
- 넓은 고정형 데스크톱 레이아웃을 우선한다.
- 표 기반 편집이 주 인터랙션이다.

## 모션 규칙

- 전역 전환은 `opacity + y` 수준으로 유지
- blur, scale, height 애니메이션 재도입 금지에 가깝게 신중히 판단
- GPU 비용이 큰 장식성 효과를 늘리지 않는다

## 시각 금지 패턴

- 과한 배경 노이즈
- 큰 오브 장식 남발
- glass 패널에 무거운 backdrop-filter
- body 전역 background/color 전환
- 의미 없이 많은 보조 설명 텍스트

## UI 변경 전 질문

- 같은 의미를 아이콘과 상태 표현만으로 전달할 수 있는가
- 설명 문장을 삭제해도 사용 흐름이 유지되는가
- 현재 액션이 표면 위에 충분히 드러나는가
- 이 효과가 실제 체감 가치를 주는가
