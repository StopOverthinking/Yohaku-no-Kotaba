# Learn Mode

## 목적

선택한 단어들을 반복 라운드로 학습하는 기본 카드 학습 모드다.

## 진입점

- 설정: `/learn`
- 세션: `/learn/session`
- 결과: `/learn/result`

## 핵심 상태

- 설정 기본값: `preferencesStore.learnDefaults`
- 세션: `learnSessionStore.record`
- 결과: `learnSessionStore.lastResult`
- 즐겨찾기: `favoritesStore.favoriteIds`
- 시험 오답 세트 가능: `examStore.wrongAnswerIds`

## 설정 규칙

### 세트 선택

- `all`
- `favorites`
- `wrong_answers`
- 기본 세트와 주제형 단어장만 선택 가능

### 필터

- 즐겨찾기만
- 범위 지정
- 범위 시작/끝 입력은 편집 중 빈 칸을 허용하고, 빈 값으로 편집을 끝내면 편집 전 수치로 복구
- `반드시 포함할 범위` 토글
- 필수 범위는 여러 구간을 목록으로 추가·삭제할 수 있으며 각 시작/끝 입력도 편집 중 빈 칸과 blur 복구를 지원
- 필수 범위는 현재 단어장과 즐겨찾기 조건을 적용한 목록의 순번을 기준으로 계산
- 겹치는 필수 범위의 단어는 한 번만 세고, 일반 범위 밖에 있더라도 세션에 포함
- 학습 항목 수는 실제 필수 단어 수보다 작아질 수 없으며, 부족하면 필수 단어 수로 자동 보정
- 앞면 기준 `japanese | meaning`
- 학습 항목 수

실제 후보 추출은 `src/features/study/wordSelection.ts`가 담당한다.

## 세션 흐름

1. 현재 설정으로 일반 후보와 필수 범위 후보를 다시 만든다.
2. 필수 범위 후보를 모두 먼저 확보한다.
3. `wordCount - 필수 단어 수`만큼 일반 후보에서 중복 없이 무작위로 채운다.
4. 완성된 세션 후보 전체를 다시 섞는다.
5. 맞음/모름으로 현재 카드를 처리한다.
6. 모르는 카드는 `retryQueue`에 들어간다.
7. 현재 라운드가 끝나면 `retryQueue`만으로 다음 라운드를 만든다.
8. 더 이상 재시도 카드가 없으면 완료한다.

설정 화면에서 계산한 후보 수는 표시용이며, 같은 단어장과 같은 개수로 다시 시작해도 이전 세션의 선택 묶음을 재사용하지 않는다.

## Undo 규칙

- 단일 이전 상태가 아니라 `snapshotHistory` 스택 사용
- 여러 번 연속 undo 가능
- undo 후에도 세션은 다시 `localStorage`에 저장

## 저장 규칙

- 진행 중 세션만 `localStorage`
- 완료 결과는 메모리 상태만 사용
- 세션 시작, 진행, undo 시 저장
- 카드 전환 애니메이션 중 백그라운드로 가면 pending 판정을 즉시 반영해 저장
- 완료 시 세션 키 삭제

## UI 규칙

- 설정 화면은 빠른 선택 중심
- 필수 범위는 토글 아래의 간결한 행 목록으로 편집하고 추가·삭제는 아이콘 버튼으로 제공
- 완료 화면은 홈 이동 아이콘 버튼 1개만 유지
- 세션 파기는 아직 브라우저 confirm을 쓰지만 향후 앱 내부 모달로 교체 예정

## 수정 시 주의점

- 선택 로직은 시험 오답 세트와 연결된다
- 비교형 단어장과 비교형 오답 기록은 이 모드에 다시 섞지 않는다
- 세션 엔진과 페이지 전환 애니메이션이 함께 움직인다
- 완료 결과를 영속 저장 대상으로 확대하지 않는다
