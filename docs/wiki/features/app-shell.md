# App Shell

## 목적

메인 앱의 공통 프레임, 라우팅, 홈 진입 구조를 설명한다.

## 진입점

- 메인 부팅: `src/main.tsx`
- 라우터: `src/app/router.tsx`
- 셸: `src/app/App.tsx`
- providers: `src/app/providers.tsx`
- 홈: `src/features/home/HomePage.tsx`

## 핵심 상태

- 라우터는 `createBrowserRouter` 기반
- `AppProviders`가 앱 시작 시 여러 스토어 `hydrate()`를 호출
- 홈은 각 세션 스토어의 현재 상태를 읽어 배너를 표시

## 라우트 분리

- 메인 앱은 `/` 하위 중첩 라우트
- 에디터는 `/editor` 단독 라우트

이 분리는 학습 앱과 데이터 관리 앱의 성격 차이를 반영한다.

## 라우트 전환 규칙

- 효과 축소 조건이면 즉시 교체
- 일반 조건이면 현재 화면이 짧게 exit 후 다음 화면 enter
- 전환 효과는 `opacity + y`만 사용

## 홈 구조

홈은 세 가지 역할을 가진다.

- 잔여 세션/결과를 배너로 복구
- 목록/학습/공유의 1차 허브
- 학습 하위 메뉴의 2차 허브

## 홈에서 바로 보여주는 상태

- 진행 중 일반 학습
- 진행 중 시험
- 최근 시험 결과

최근 시험 결과 배너는 결과 화면으로 이동하거나 결과 기록만 삭제할 수 있다.
삭제해도 시험 오답 노트 ID는 유지한다.

## UI 규칙

- 홈 카드 설명은 짧게
- 중요한 이동은 큰 카드
- 서브메뉴는 필요할 때만 펼침
- compact 환경에서도 같은 정보 구조 유지

## 수정 시 주의점

- 새 feature를 추가하면 라우터와 홈 진입 지점을 함께 조정
- 새로운 스토어가 hydrate 필요하면 `AppProviders`에 추가
- 전역 애니메이션을 무겁게 만들지 말 것
