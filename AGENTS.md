# Codex Start Here

## 위키를 먼저 읽는다

코드를 검색하거나 열기 전에 아래 순서를 반드시 따른다.

1. `docs/wiki/INDEX.md`
2. `docs/wiki/llm-workflow.md`
3. 작업 대상 기능 문서

문서가 있으면 문서를 먼저 읽고, 코드 확인은 그 다음이다.

## 위키 운영 규칙

- 구현을 바꾸면 관련 위키도 함께 갱신한다.
- 위키와 코드가 다르면 최종 상태에 맞게 둘 다 정리한다.
- `탭 매치 러시`는 아직 미완이므로 위키 범위에서 제외한다.
- 관련 코드가 있어도 설계 기준이나 리팩터링 기준으로 삼지 않는다.

## 프로젝트 UI 원칙

- 언제나 미니멀리즘 디자인, 아이콘 중심 디자인을 우선시한다.
- 텍스트 설명은 최소화한다.
- 사용자가 설명 없이도 스스로 기능의 의미를 알 수 있다고 가정한다.
- 고비용 blur, filter, height animation, backdrop-filter를 다시 늘리지 않는다.

## 에디터 원칙

- 단어장 관리용 에디터 앱은 모바일 사용을 고려하지 않는다.
- 에디터는 데스크톱 기준의 넓은 고정형 레이아웃을 우선한다.
- 선택 상태와 주요 조작은 표 자체에서 바로 드러나야 하며 별도 우측 상세 패널에 의존하지 않는다.
